# Security Policy

## Reporting a Vulnerability

We take the security of SpecInferKit seriously. If you discover a security vulnerability, please do **not** open a public issue.

Instead, send a private report to the project maintainers via GitHub's private vulnerability reporting at:
https://github.com/Crynge/SpecInferKit/security/advisories/new

## What to Include

- Type of vulnerability
- Steps to reproduce
- Potential impact
- Suggested fix (if applicable)

## Response Timeline

- **72 hours**: Acknowledgment of receipt
- **7 days**: Initial assessment and plan
- **30 days**: Fix released (for confirmed vulnerabilities)

## Supported Versions

| Version | Supported |
|---------|-----------|
| >= 0.1.0 | Yes |
| < 0.1.0 | No |
